std

i found out from an early time in my coding career that i had a passion for building hacky tools and what not. this passion led me to my first project i ever built!

std, which stands for shit to do, was a really fun and simple project that i have continuously gotten to hack on throughout my humble career in software.

std is a way to manager all of your tasks using vim and some simple bash setup. all i did for my setup was run the executable by creating an alias in my bash profile!

i got to build this project purely in go, and was able to expand on the original idea by exploring some of go's community-built frameworks.

specifically i got to understand how databases work by using boltdb, which is an embedded, key value database written purely in go! you can check out how i used their database in an example here.

i also got to build out the cli with a fun framework called cobra. you can see the code for that here.

the project has become a center for my workflow both at school, on personal projects, and at my positions as a software engineer. all in all, a very humbling and exciting first project!




rtc

this was another really fun project that helped me solidify my passion for tooling.

rtc stands for rescuetime cli. rescuetime is an application that allows for the tracking of your time you spend on your laptop.

i found it counter-intuitive to have to leave the command line to see how you were doing on your productivity for that day, so i decided to build this in order to fix that issue and keep you in your terminal longer!

i got to build this tool using go, and again was able to expand on the project by using my two favorite frameworks: boltdb and cobra!

i really focused this project on my ability to get something tangible to work quickly and efficiently. the final product is something that i am really happy with for right now, and am really excited to expand on it soon!
